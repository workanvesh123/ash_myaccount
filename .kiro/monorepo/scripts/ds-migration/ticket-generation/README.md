## Design System Migration Pipeline

Fast, dependency-free scripts for generating design system migration tickets from Code PushUp reports. Fully automated pipeline with comprehensive validation and quality assurance.

### Quick Start

**Recommended: Use Kiro Hook (Automated)**
```
Open Command Palette → Search "Kiro Hook" → Run "Generate & Group DS Migration Tickets"
```
This runs the complete pipeline automatically with progress tracking and error handling.

**Manual: Run Scripts Directly**
```bash
# Run complete pipeline
node run-migration-pipeline.mjs

# Group tickets by project (after pipeline completes)
node group-all-projects.js

# Group specific project tickets
node group-tickets.js .code-pushup/.tickets/poker-core-lib/tickets

# With custom paths (optional)
node run-migration-pipeline.mjs [reports-path] [ui-path] [stories-path]

# Merge two migration tickets (utility)
node internal/05-merge-migration-tickets.mjs <ticket1.json> <ticket2.json> [output.json]
```

### Architecture

**4-Step Pipeline:**
1. **Extract** - Parse Code PushUp JSON reports for component usage (11,522 occurrences)
2. **Pre-compute** - Generate metadata and find implementation examples (34 components)
3. **Generate** - Create structured migration tickets per project/feature (601 tickets)
4. **Validate** - Ensure all tickets pass validation and quality checks (100% success)

### Scripts

**Main Pipeline:**
- `run-migration-pipeline.mjs` - Orchestrates complete pipeline (4 steps)
- `group-all-projects.js` - Groups tickets for all projects with generated tickets
- `group-tickets.js` - Groups tickets for a specific project by feature hierarchy

**Internal Steps:**
- `internal/01-generate-ds-usage-occurrences.mjs` - Extract usage from reports
- `internal/02-precompute-implementation-data.mjs` - Pre-compute metadata and examples  
- `internal/03-ds-migration-ticket-generator.mjs` - Generate final tickets
- `internal/04-validate-migration-tickets.mjs` - Validate all generated tickets
- `internal/05-merge-migration-tickets.mjs` - Merge two migration tickets into one

**Validation & Schema:**
- `internal/shared-validation.mjs` - Common validation logic
- `internal/migration-ticket.schema.json` - JSON schema for ticket validation

**Kiro Hook:**
- `.kiro/hooks/ds-migration-tickets.kiro.hook` - Automated pipeline with grouping

### Ticket Merging Utility

**Merge Migration Tickets:**
```bash
node internal/05-merge-migration-tickets.mjs <ticket1.json> <ticket2.json> [output.json]
```

**Features:**
- **Smart Merging**: Combines design system components from both tickets
- **Deduplication**: Removes duplicate project implementations based on path and lines
- **Statistics Recalculation**: Updates all occurrence and implementation statistics
- **Validation**: Ensures merged ticket passes schema validation
- **Same Project Only**: Validates tickets are from the same project before merging

**Use Cases:**
- Consolidate related feature tickets into larger migration units
- Combine tickets that share common components or implementations
- Create comprehensive migration tickets for complex features
